# Token Efficiency & Execution Rules
- Think deeply but keep final text outputs strictly concise and code-only where possible.
- Never rewrite an entire file. Only output targeted, minimal diffs/edits.
- Do not explain basic syntax or give conversational filler ("Sure, I can help with that").
- Read a file exactly once per session unless it has been explicitly modified.
- Always run tests locally before declaring a task finished to prevent diagnostic loops.
