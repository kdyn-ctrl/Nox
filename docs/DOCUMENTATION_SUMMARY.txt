================================================================================
WHAT'S COVERED VS. NOT COVERED
================================================================================

COVERED (Comprehensive Testing)
✅ Core logic (RegimeStateMachine, Kelly sizing, MCPT)
✅ All validation gates (schema, auth, momentum, regime, portfolio)
✅ External integrations (Alpaca, Yahoo Finance, Telegram)
✅ Error handling (timeouts, invalid inputs, missing data)
✅ User alerts (all Telegram message types)
✅ Performance metrics (win rate, Sharpe ratio, P&L)

NOT COVERED (Acceptable)
❌ Mocked APIs (tested against real paper trading instead)
❌ Load testing (system processes one signal at a time)
❌ Fuzzing (system has strict input gates)
❌ Backwards compatibility (maintained by one person)

================================================================================
MAINTENANCE SCHEDULE
================================================================================

Monthly (30 min): Spot-check one random guide
Quarterly (1 hour): Review "Last verified" dates, update old ones
When bug found: Add test case to guide immediately
When code changes: Update guides before committing

Red flags that guide needs updating:
- Example doesn't compile
- Expected output doesn't match
- Threshold in guide ≠ threshold in code
- Env var in guide doesn't exist
- Alert message different from actual

HOW TO USE THIS DOCUMENTATION SYSTEM
================================================================================

Starting a Feature:

  4. Approve if docs match code

Regular Maintenance:
  1. Run monthly spot check
  2. If something is wrong, fix it immediately
  3. Update timestamp when verified
  4. Keep guides from decaying
